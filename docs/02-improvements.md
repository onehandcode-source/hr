# HR 시스템 - 개선 사항 및 추가 기능 요구사항

## 1. 버그 및 기술적 개선

### 1-1. 인증 보안
- [ ] **비밀번호 정책 미적용**: 현재 최소 길이·특수문자 제한 없음 → 정책 강화 필요
- [ ] **세션 만료 처리 부재**: 토큰 만료 시 자동 로그아웃 + 안내 메시지 추가 필요
- [ ] **로그인 시도 제한 없음**: Brute Force 공격 방어를 위한 횟수 제한(Rate Limiting) 필요

### 1-2. 데이터 유효성 검사
- [ ] **API 입력값 검증 부족**: 서버 사이드에서 Zod 등을 활용한 스키마 검증 적용 필요
- [ ] **휴가 신청 날짜 중복 체크 미흡**: 동일 날짜 중복 신청 방지 로직 강화 필요
- [ ] **연차 잔여일 음수 방지**: 잔여일 초과 신청 시 명확한 오류 반환 필요

### 1-3. 에러 핸들링
- [ ] **API 에러 응답 불일치**: 에러 코드·메시지 형식이 엔드포인트마다 다름 → 표준화 필요
- [ ] **클라이언트 에러 바운더리 없음**: React Error Boundary 미적용으로 전체 화면이 크래시될 수 있음
- [ ] **미처리 Promise Rejection**: 일부 API 호출에 try/catch 누락

### 1-4. 성능
- [ ] **React Query 캐시 전략 없음**: staleTime, cacheTime 설정으로 불필요한 API 재호출 줄이기
- [ ] **이미지 최적화 미적용**: `<img>` 태그 사용 → Next.js `<Image>` 컴포넌트로 교체
- [ ] **대용량 데이터 페이지네이션 없음**: 직원·휴가 목록이 많아질 경우 성능 저하 우려

---

## 2. UX / UI 개선

### 2-1. 사용성
- [ ] **모바일 반응형 미완성**: 일부 테이블·캘린더가 모바일에서 레이아웃 깨짐
- [ ] **로딩 상태 일관성 부재**: 스켈레톤 UI vs 스피너가 혼재, 통일 필요
- [ ] **빈 상태(Empty State) 처리**: 데이터 없을 때 안내 문구·이미지 부재
- [ ] **폼 유효성 피드백 개선**: 오류 시 해당 필드 하이라이트 및 구체적인 메시지 제공

### 2-2. 접근성 (Accessibility)
- [ ] **ARIA 레이블 부족**: 스크린 리더 지원을 위한 aria-label·role 속성 추가 필요
- [ ] **키보드 내비게이션**: 모달·드롭다운의 포커스 트랩 미구현
- [ ] **색상 대비율**: 일부 텍스트가 WCAG AA 기준 미달 가능성

## 3. 추가 기능 요구사항

### 3-1. 휴가 관리 확장
- [ ] **공휴일 자동 제외**: 현재 공휴일 유틸이 있으나 휴가 일수 계산에 완전히 반영되지 않음
- [ ] **연차 자동 부여**: 연도별·입사일 기준 연차 자동 충전 스케줄러 필요
- [ ] **휴가 유형 확장**: 경조사 휴가, 육아휴직, 출산휴가 등 추가 유형 필요
- [ ] **휴가 신청 취소 플로우**: 승인된 휴가 취소 시 관리자 재승인 프로세스 필요
- [ ] **휴가 통계 리포트**: 월별·부서별 휴가 사용 현황 엑셀/PDF 내보내기

### 3-2. 인사 평가 고도화
- [ ] **평가 주기 자동화**: 반기·연간 평가 기간 설정 및 알림 발송
- [ ] **다면 평가(360도)**: 동료·하급자 평가 기능 추가
- [ ] **평가 결과 비교**: 기간별 점수 추이 차트 제공
- [ ] **평가 항목 가중치**: 항목별 가중치 설정으로 총점 자동 계산
- [ ] **평가 승인 워크플로우**: 평가 제출 → 검토 → 확정 단계 구분

### 3-3. 알림 시스템 강화
- [ ] **이메일 알림**: 휴가 승인·반려 시 이메일 발송 (Nodemailer / Resend)
- [ ] **실시간 알림**: Polling 방식 → WebSocket 또는 SSE(Server-Sent Events)로 개선
- [ ] **푸시 알림**: PWA 기반 브라우저 푸시 알림 지원
- [ ] **알림 설정**: 사용자별 알림 수신 방식 선택 기능

### 3-4. 관리자 기능 확장
- [ ] **부서 관리**: 부서 추가·수정·삭제 (현재 하드코딩)
- [ ] **직급 관리**: 직급 체계 관리 기능
- [ ] **권한 세분화**: 부서장, HR 담당자 등 중간 권한 역할 추가 (RBAC 확장)
- [ ] **감사 로그(Audit Log)**: 주요 데이터 변경 이력 추적
- [ ] **대량 작업**: 직원 일괄 등록(CSV 업로드), 일괄 연차 조정

### 3-5. 보고서 및 통계
- [ ] **인사 대시보드 고도화**: 이직률, 결근율, 평가 평균 등 KPI 지표 추가
- [ ] **월별·분기별 리포트**: 자동 생성 및 PDF 다운로드
- [ ] **부서별 비교 분석**: 부서 간 휴가 사용·평가 점수 비교

### 3-6. 시스템 운영
- [ ] **다국어 지원(i18n)**: 현재 한국어 전용 → next-intl 적용으로 확장성 확보
- [ ] **다크 모드**: 현재 라이트 모드만 지원
- [ ] **환경 설정 관리**: 관리자가 UI에서 시스템 설정 변경 가능하도록
- [ ] **백업 및 복원**: DB 정기 백업 스케줄 및 복원 기능

## 4. 개발 환경 및 코드 품질

- [ ] **테스트 코드 부재**: Unit Test (Vitest), E2E Test (Playwright) 전혀 없음
- [ ] **CI/CD 파이프라인**: GitHub Actions를 통한 자동 빌드·테스트·배포 미구성
- [ ] **환경 변수 타입 안전성**: `.env.local` 미검증 → `@t3-oss/env-nextjs` 적용 권장
- [ ] **API 문서화**: Swagger / OpenAPI 스펙 없음
- [ ] **Storybook**: UI 컴포넌트 문서화 및 독립 개발 환경 부재
- [ ] **로깅 시스템**: 서버 사이드 구조화 로깅 없음 (Pino / Winston 적용 권장)
